Released 2026-04-18 from `master` → tag `v4.4.0`.

55 commits across 4 arcs in one sitting. Headline: **mainstream wallets can now send ETH to `*.whelm.eth` addresses that resolve to IKA-custodied cross-chain squids.** Under the hood, the Sneasel + Weavile infra makes those sends privacy-preserving once the UI hook + 4337 paymaster gates land.

## Highlights

### 1. `*.whelm.eth` is live ENS CCIP-read on Ethereum mainnet

Full Beldum → Metagross evolution shipped in 18 moves. Gateway serves any SUIAMI-registered identity under `brando.whelm.eth`, `superteam.whelm.eth`, etc., with proper wildcard resolution. Mainstream wallets (Brave, Rainbow, Rabby, MetaMask, Coinbase Wallet, Phantom itself) resolve these names. PayPal / Venmo / Google Pay's "send to name.eth" flows all reach these identities.

On-chain artifacts:
- **OffchainResolver contract** `0x82a75f7bf6a92207dc15b52fc266855382d8334d` (Ethereum mainnet) — EIP-3668 CCIP-read, SignatureVerifier-gated, admin-rotatable
- **whelm.eth resolver bound** — wrapped owner delegated to eth@ultron via NameWrapper setApprovalForAll; future rotations threshold-gated
- **CCIP-read gateway** — `https://sui.ski/ens-resolver/{sender}/{data}.json` serves signed responses from the `ENS_SIGNER_PRIVATE_KEY` hot key with eth@ultron as standing co-signer

Ceremony hits on the way (all in this release):
- Brave Wallet + Me Wallet + peer.xyz provider-hijacking solved via EIP-6963 picker
- Phantom's strict param validator satisfied (always switch chain + minimal tx params)
- NameWrapper auto-detect for wrapped names
- Delegate-to-Ultron approval so black-diamond owner keys stay dormant

### 2. Sneasel — private-send infrastructure (gated, not yet live)

11 moves shipped for per-counterparty guest subnames (`amazon.guest.sui`) that resolve to Seal-encrypted, ultron-brokered hot addresses. Architecture:

- **Move module** — `GuestStealthKey` + `GuestStealth` + `seal_approve_guest_stealth` on SUIAMI roster (package v8 at `0xf4910af0…e915c77`)
- **Client flow** — `buildGuestPrivateTx` Seal-encrypts cold destinations against a policy only ultron (the sweep_delegate) can satisfy
- **SneaselWatcher DO** — scaffold for webhook-driven sweep hot → (per-guest intermediate) → cold
- **Ice Fang** — sweep-graph collapse fix: distinct cold destinations per guest + per-guest intermediate dWallet, no cross-guest batching
- **Icy Wind** — real per-guest IKA DKG ceremony (`provisionDWallet`) replaces the Ice Fang stub

**Not live to users yet:** gated behind threat-model-compliant UI language (see `docs/superpowers/plans/2026-04-18-sneasel-weavile-threat-model.md`) and Assurance (4337 paymaster). Calling Sneasel "private" before Assurance ships is a bug.

### 3. Weavile — multi-chain stealth scaffold (also gated)

3 scaffold moves opening #198. Where Umbra/Fluidkey/Railgun are ETH-only, Weavile uses IKA 2PC-MPC cross-curve to support ETH + BTC + Sui + Solana + Tron + any L2 from day one.

- **Move module** — `StealthMetaKey` + `set_stealth_meta` entry + `stealth_announcer::announce` for Sui/Sol native event-based announcements (EIP-5564 analog)
- **Meta-address format** — `ska:<ika_dwallet_id>:<chain=view_pubkey>|...` published at `brando.whelm.eth` ENS text record
- **Client helpers** — `generateViewKeys` (secp256k1 + ed25519), `serializeMetaAddress`/`parseMetaAddress`, `buildSetStealthMetaTx`, `publishStealthMeta()` browser console hook
- **Scanner architecture** — `src/server/agents/weavile-scanner.md` spec doc for Pursuit (not-yet-built DO that scans Announcer events and derives matching stealth keys via view key)

Core differentiator: **keyless recipient + gasless sweeps + multi-chain**. Stock stealth-addr protocols can't offer this without IKA.
